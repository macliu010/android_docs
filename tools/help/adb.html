  1. 模拟器

Android Debug Bridge (adb)是一个多功能的命令行工具，能让你与一个模拟器或者已连接的android设备交互。它是一个client-server程序，包含三个组件：

  • client，运行在你的开发机上。你可以通过发出adb的shell命令来调用一个客户端。其他的android工具，诸如ADT插件和DDMS同样可以创建adb客户端。
  • server，作为后台进程运行在你的开发机上。它管理客户端与运行在模拟器或设备上的adb daemon之间的通信。
  • daemon，作为后台进程运行在每一个模拟器或设备实例上。

你可以查找adb工具在<sdk>/platform-tools/.

当你开启一个adb client时，客户端首先检查是否有一个已经运行的adb server进程。如果没有，将开启server进程。当服务器启动时，将绑定本地5037TCP端口，监听从adb client发送的命令—所有adb client使用5037端口与adb server通信。

然后，server与所有正在运行的模拟器/设备实例建立连接。通过扫描在5555到5585范围内的奇数端口上的模拟器/设备实例，这个端口范围被模拟器/设备使用。server在哪个端口上查找到adb daemon，就在该端口上建立连接。注意，每一个模拟器/设备实例获得一对顺序端口—一个偶数端口的console连接和一个奇数端口的adb连接。例如：

模拟器 1, console: 5554
模拟器 1, adb: 5555
模拟器 2, console: 5556
模拟器 2, adb: 5557 ...

如上所述，adb在5555端口上连接到模拟器实例，与console监听在5554上的是一个实例。

一旦服务器与所有模拟器实例建立完了连接，你可以使用adb命令控制和访问那些设备。由于server管理连接到模拟器/设备实例和处理来自许多adb client的命令，所以你可以从任何一个client（或者一个script）来控制模拟器/设备实例。

下面部分描述了你可以使用命令访问adb功能和管理一个模拟器/设备的状态。注意，如果你正在使用已经安装了ADT插件的Eclipse开发android应用，你不需要从命令行访问adb。ADT插件提供了一个集成到Eclipse IDE的adb，对你来说是透明的。然而，你仍然可以在必要时直接使用adb,比如debugging。

发出ADB命令

你可以从开发机上的命令行或者一个script中输入adb命令。用法如下： 

adb [-d|-e|-s <serialNumber>] <command>

当你输入一个命令时，程序调用一个adb client。client没有与特定的模拟器实例相关联，因此如果有许多的模拟器/设备在运行，你需要使用-d选项来指派命令去指定的目标实例。关于使用选项的更多信息，查看指派命令到指定的模拟器/设备实例

查询模拟器/设备实例

在发出adb命令之前使用，有助于了解adb server连接了哪些模拟器/设备实例。你可以使用devices命令生成一个模拟器/设备的列表： 

adb devices

在响应中，adb打印出每个实例的状态信息：

  • 序列号 — adb通过使用控制端口号创建一个字符串来唯一标识一个模拟器/设备实例。Serial number的格式是 <type>-<consolePort>. 一个serial number的例子： emulator-5554(译者注：实际usb连接设备的格式有所不同，例如0149A45511013013)
  • 状态 — 实例的连接状态可能如下：
    • offline — 实例没有被建连到adb或者没有反应（译者注：当模拟器在开启时也显示offline）。
    • device — 实例与adb server正在连接中。注意，这个状态不意味android系统是完全启动和运行的，因为实例连接到adb时系统仍然在启动期间。然而，启动之后，进入模拟器/设备实例的正常运行状态。
    • no device — 没有接入的模拟器/设备。
    • (译者注：no permissions — 设备已连接到adb，但是没有授权，具体请参考Using Hardware Devices)。

输出每个实例的格式如下： 

[serialNumber] [state]

下面例子显示了devices命令以及输出: 

$ adb devices
List of devices attached 
emulator-5554  device
emulator-5556  device
emulator-5558  device

指派命令到指定的模拟器/设备

如果有许多的模拟器/设备正在运行，在发出adb命令时你需要指定一个目标实例。在命令行中使用-s选项。选项-s的用法如下： 

adb -s <serialNumber> <command>

如上所示，你通过命令中使用adb为它分配的序列号来指定目标实例。你可以使用devices命令获得正在运行的模拟器/设备实例的序列号。

例子如下： 

adb -s emulator-5556 install helloWorld.apk

注意，如果你输入一个使用了-s的命令，但是没有指定目标模拟器/设备实例，adb会产生一个error。

安装应用

你可以使用adb从你的开发电脑上复制一个应用，并且把它安装到一个模拟器/设备实例。可以这样做，使用install命令。用这个命令时，你必须指定想要安装的.apk文件的路径，如下： 

adb install <path_to_apk>

获得更多有关如何创建一个可以安装在一个模拟器/设备上的.apk文件的信息，可以查看Building and Running

注意，如果你使用已经安装了ADT插件的Eclipse IDE，在模拟器/设备上你不必直接使用adb（或者aapt）安装你的应用。反而，使用ADT插件为你处理应用的打包和安装。

转发端口

你可以使用forward命令去设备任意端口的转发 — 将指定主机端口的请求转发到一个模拟器/设备实例的另一个端口。以下告诉你如何设置将主机6100端口的请求转发到模拟器/设备的7100端口： 

adb forward tcp:6100 tcp:7100

你也可以使用adb设置将请求转发到抽象的UNIX domain sockets命名，如下：

adb forward tcp:6100 local:logd

从模拟器/设备中拷入或拷出文件

你可以使用adb命令的pull和push，复制文件到模拟器/设备上，或者从模拟器/设备上复制出来数据文件。不同与install命令，仅仅是复制一个.apk文件到指定位置，pull和push命令让你复制任意的目录和文件到模拟器/设备实例的任何位置。

从模拟器或设备上复制一个文件或者文件夹（递归），使用如下： 

adb pull <remote> <local>

复制一个文件或文件夹（递归）到模拟器或设备上，使用如下： 

adb push <local> <remote>

在命令中，<local>和<remote>指的是开发机（本地）的路径到模拟器/设备实例（远程）的目标文件/文件夹，或者模拟器/设备实例（远程）的路径到开发机（本地）目标文件/文件夹。

例子如下：

adb push foo.txt /sdcard/foo.txt

发出Shell命令

adb提供了一个ash shell，你可以使用ash shell在模拟器或设备上运行多种命令。这些命令以二进制的形式保存在模拟器或设备的本地文件系统中，如下： 

/system/bin/...

不论进没进入到模拟器/设备的adb远程shell终端，你都可以使用shell命令去发布命令。

当没有进入远程shell终端时，发布一个简单的shell命令，如下： 

adb [-d|-e|-s {<serialNumber>}] shell <shellCommand>

使用shell命令登录进一个远程的模拟器/设备的shell终端。如下： 

adb [-d|-e|-s {<serialNumber>}] shell

当你准备退出远程shell时，使用CTRL+D或exit来关闭shell会话。

以下将告诉你更多的关于使用shell命令的知识。

通过远程shell连接sqllite3数据库

通过adb远程shell，你可以使用sqlite3客户端程序去管理android应用中安装的SQLite数据库。sqlite3工具包括很多有用的命令，例如.dump导出一个table的内容，.schema导出现存table的SQL CREATE报表。Sqlite3还允许你远程执行SQLite命令。

使用sqlite3，按照前面章节的介绍，进入到模拟器的远程shell终端，然后使用sqlite3命令启动工具。当启动sqlite3后，你可以指定你想查看的数据库的完整路径。模拟器/设备实例会在文件夹/data/data/<package_name>/databases/中保存SQLite3数据库。

示例如下：

$ adb -s emulator-5554 shell
# sqlite3 /data/data/com.example.google.rss.rssexample/databases/rssitems.db
SQLite version 3.3.12
Enter ".help" for instructions
.... enter commands, then quit...
sqlite> .exit

当你想要启动sqlite3时，你可以在shell中发布sqlite3命令。使用exit或CTRL+D命令可以退出和返回到adb远程shell终端。

UI/Application试验程序 Monkey

当Monkey程序在模拟器或设备运行的时候，如果用户产生了比如点击，触摸，手势或一些系统级别的事件的时候，它就会产生pseudo-randomstreams。你可以用Monkey采用随机重复的方式去stress-test你开发的应用程序。

最简单的方法就是用下面的命令来使用Monkey，这个命令将会启动你的应用程序并且触发500个pseudo-random事件。 

$ adb shell monkey -v -p your.package.name 500

更多的关于命令Monkey的命令的信息，可以查看UI/Application Exerciser Monkey文档页面。

其他shell命令

下表列出了一些有用的adb shell命令。如果需要全部的命令和程序，可以启动模拟器实例并且用adb -help命令。 

adb shell ls /system/bin

对大部分命令来说，help都是可用的。

Shell命令 描述 注释
dumpsys dump出系统信息数据。 The Dalvik Debug Monitor Server (DDMS) tool offers integrated debug environment that you may find easier to use.
dumpstate dump出状态到文件中。
logcat [<option>]... [<filter-spec>]... Enables radio logging and prints output to the screen.
dmesg 输出核心的debugging信息到屏幕上
start 开启（重启）一个模拟器/设备实例。  
stop 关闭一个模拟器/设备实例。  

启用logcat日志

Android日志系统提供了收集和查看系统debug信息的机制。日志是从各个应用程序和一些系统程序中收集,保存到一个连续循环的buffer，可以通过logcat命令来查看和过滤信息。

你可以使用logcat命令查看和跟踪系统log缓冲区的内容。用法如下： 

[adb] logcat [<option>] ... [<filter-spec>] ...

你可以在开发机或模拟器/设备的远程adb shell终端上使用logcat命令。将log输出到你的开发机上，如下： 

$ adb logcat

从远程adb shell终端 

# logcat

查看Reading and Writing Logs 获得关于logcat命令的操作与过滤规则的完整的信息。

关闭adb server

在某些情况下，你可能需要终止adb server进程，然后再重启它。例如,adb没有响应一个命令，也许通过终止并且重启serrver，能解决这个问题。

用kill-server可以终止adb server。你可以发出任何adb命令来重新启动server。
